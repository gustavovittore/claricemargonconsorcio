# PREMIUM UI RULES — CLARICE MARGON CONSÓRCIOS

## OBJETIVO DESTE DOCUMENTO

Este arquivo define as regras visuais e de interface para a nova versão da landing page de Clarice Margon.

A primeira versão criada ficou genérica, sem força visual, com copy fraca, estrutura pouco comercial e estética distante da referência desejada.

A nova versão deve ser reconstruída com uma direção mais próxima da landing page do Ezequiel do Consórcio, porém adaptada para Clarice Margon com identidade própria, estética premium, feminina, elegante, clara e consultiva.

A página precisa ter aparência:

- premium;
- moderna;
- comercial;
- refinada;
- confiável;
- bem diagramada;
- centralizada no miolo;
- visualmente forte;
- responsiva;
- com ritmo visual.

O objetivo é evitar qualquer aparência genérica, amadora ou de template barato.

--------------------------------------------------

# PRINCÍPIO CENTRAL

Toda seção deve responder a uma pergunta:

"Isso aumenta a percepção de confiança, autoridade, clareza e desejo de falar com a Clarice?"

Se a resposta for não, o elemento deve ser removido ou refinado.

A página deve vender bem, mas sem parecer agressiva.

Ela deve parecer uma consultoria financeira premium com força comercial.

--------------------------------------------------

# PALETA OFICIAL

A landing page deve seguir esta paleta principal:

- Fundo principal: `#F2F0EF`
- Textos principais: `#1F1F1F`
- Destaques, botões e detalhes: `#E19D35`

A combinação deve transmitir:

- sofisticação;
- patrimônio;
- estabilidade;
- elegância;
- clareza;
- autoridade;
- confiança;
- aparência premium.

O dourado deve ser usado com moderação.

Usar dourado em:

- botões principais;
- detalhes finos;
- números importantes;
- ícones selecionados;
- linhas decorativas;
- pequenos destaques;
- elementos de autoridade;
- hover de botões/cards.

Evitar dourado em excesso.

--------------------------------------------------

A interface deve ser:

- moderna;

- assessoria em consórcios;
- wealth management;
- página comercial de alto padrão;
- imobiliário premium;

A página NÃO deve lembrar:

- landing page genérica;
- página comum de consórcio;
- dashboard financeiro;

- página fria demais;
- template pronto;

--------------------------------------------------

# REGRA DE LAYOUT E CONTAINER

A página deve ter conteúdo concentrado no miolo.

A primeira versão ficou visualmente solta e com pouca força de composição.

A nova versão deve usar:

- container centralizado;
- largura máxima controlada;
- margens laterais generosas;
- alinhamento preciso;
- seções bem conectadas;
- blocos com ritmo;
- respiro visual intencional;
- nada encostado nas bordas.

## Direção prática

No desktop:

- não deixar textos começando muito perto da lateral esquerda;
- não deixar elementos espalhados demais;
- manter tudo dentro de um grid consistente;
- usar containers com largura máxima elegante;
- criar sensação de página bem diagramada.

No mobile:

- reduzir margens sem apertar;
- manter textos confortáveis;
- evitar que títulos quebrem feio;
- preservar hierarquia;
- garantir botões fáceis de tocar.

--------------------------------------------------

# O QUE EVITAR

Nunca criar:

- layout genérico;
- visual poluído;
- excesso de centralização sem intenção;
- excesso de elementos flutuantes;
- cards apertados;
- excesso de ícones;
- excesso de efeitos;
- sombras exageradas;
- animações chamativas;
- gradientes fortes;
- elementos sem função;
- textos gigantes sem hierarquia;
- Hero fraca;
- Hero com H1 quebrando feio;
- mapa abstrato sem leitura;
- gráfico de linha fraco;
- pré-simulação interna;
- aparência de página feita por IA sem direção artística.

--------------------------------------------------

# HIERARQUIA VISUAL

Cada seção deve ter:

1. eyebrow ou pequena linha de contexto;
2. título forte;
3. subtítulo claro;
4. conteúdo visual organizado;
5. CTA quando fizer sentido.

O usuário deve entender rapidamente:

- onde está;
- o que está sendo explicado;
- por que aquilo importa;
- qual ação tomar.

A hierarquia precisa ser mais forte do que na primeira versão.

Os títulos devem parecer pensados e bem diagramados.

# BOTÕES

Botões devem ter aparência premium.

## Botão principal

Usar para WhatsApp.

Características:

- destaque forte;
- dourado elegante `#E19D35`;
- texto em grafite escuro `#1F1F1F`;
- cantos arredondados;
- hover suave;
- boa área de clique;
- aparência sofisticada.

## Botão secundário

Características:

- transparente ou claro;
- borda fina;
- texto em grafite escuro `#1F1F1F`;
- hover leve;
- aparência elegante.

Evitar:

- botões pequenos demais;
- botões com cor vibrante demais;
- efeitos exagerados;
- bordas duras;
- sombras pesadas.

--------------------------------------------------

# CARDS

Cards devem transmitir organização e sofisticação.

Regras:

- fundo claro;
- borda sutil;
- sombra suave;
- cantos arredondados;
- espaçamento interno confortável;
- ícone pequeno e refinado;
- título curto;
- texto objetivo;
- hierarquia visual.

Não criar cards com muito texto.

Não criar cards todos iguais sem hierarquia.

Pode haver cards principais com destaque maior.

--------------------------------------------------

# ÍCONES

Usar ícones apenas quando agregarem clareza.

Ícones devem ser:

- finos;
- elegantes;
- consistentes;
- preferencialmente em grafite escuro `#1F1F1F` ou dourado `#E19D35`;
- sem aparência infantil.

Evitar:

- ícones coloridos demais;
- ícones 3D genéricos;
- ícones grandes demais;
- excesso de ícones.
